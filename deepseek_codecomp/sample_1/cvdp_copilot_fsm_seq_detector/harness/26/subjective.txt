The testbench maintains a sliding window of the input stream through the use of a shift register. In the Reference Sequence Detector Logic Using Shift Register section, it explains that the shift register stores the last Length bits of the input sequence. When a new bit is received, it shifts the register such that the new bit is added to the least significant bit position, and the oldest bit is dropped. This mechanism creates a sliding window effect, allowing the reference logic to compare the stored bits with the informed sequence. The sliding window approach ensures that the entire input sequence is examined sequentially, enabling the detection of the sequence 01001110 as implemented in the DUT.
