The `call_requests_internal` variable serves as a temporary storage to keep track of pending floor requests, ensuring that each request is processed only once. When a new request arrives, it is merged into `call_requests_internal`, and any previously serviced request is cleared. This mechanism prevents duplicate stops by ensuring that each floor's request is handled exactly once before moving on to the next priority request. During movement, the system identifies the highest and lowest requested floors to determine the elevator's direction, optimizing the path towards resolving the most urgent request first. When the elevator is idle, `call_requests_internal` is reset, ensuring no pending requests interfere with subsequent operations.
