The `call_requests_internal` register maintains a bitwise representation of active floor requests. When a new request arrives via `call_requests`, it is added to `call_requests_internal` using a bitwise OR operation, ensuring that no duplicate requests are processed. When the elevator moves to a floor that has received a request, that position is cleared in `call_requests_internal` to prevent reprocessing. The system handles new requests by checking each relevant bit in `call_requests_internal` and clearing it once the request is accepted. This mechanism ensures that each request is uniquely identified and managed without duplication. During movement, the elevator prioritizes the highest or lowest remaining request based on its direction, updating `max_request` and `min_request` dynamically. When idle, all pending requests in `call_requests_internal` are cleared, ensuring efficient processing. The integration of new requests and removal of completed ones is seamless, maintaining accurate state information.
