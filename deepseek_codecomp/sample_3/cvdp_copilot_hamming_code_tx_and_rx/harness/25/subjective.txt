The testbench does not generate a fully randomized ENCODED_DATA because the receiver's functionality relies on valid encoded messages containing parity bits. By generating a DATA_WIDTH-wide input and using it to create an encoded ENCODED_DATA via golden_hamming_tx, the test ensures the encoded message is structurally correct. Modifying a single bit in this encoded data tests the receiver's ability to detect and correct errors while maintaining the integrity of the encoded format.
