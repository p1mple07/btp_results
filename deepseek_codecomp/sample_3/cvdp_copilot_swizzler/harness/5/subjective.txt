When one of the indices in the mapping_in input exceeds the expected range of N, the RTL module ignores the out-of-range index by setting the corresponding data_out lane to '0'. This occurs because the code checks if the index is within the valid range [0, N-1]; if not, it assigns '0' to data_out[i]. Therefore, it is feasible to input such out-of-range indices, and the RTL will simply set those lanes to zero.
