The testbench ensures that race conditions or timing mismatches do not occur by isolating each DUT into separate clock cycles. Each DUT is instantiated with its own configuration, allowing them to run independently without interfering with one another. The testbench uses sequential task execution for each DUT, ensuring that each is completed before moving on to the next. Additionally, the testbench monitors the out_val and done signals for each DUT separately, using cycle counters to track progress and validate results without cross-talk. By isolating each DUT in both configuration and execution, the testbench maintains integrity and accuracy across all instances.
