The testbench maintains a sliding window of the input stream through the use of the `seq_shift_reg` variable. This register is updated in each clock cycle by appending the new `seq_in` bit while shifting out the oldest bit, effectively creating a window of the past 8 bits. The reference sequence detection is performed by comparing this sliding window (`seq_shift_reg`) with the informed sequence (`informed_seq`). Whenever the sliding window matches the informed sequence, `ref_seq_detected` is set to 1, indicating successful detection.
