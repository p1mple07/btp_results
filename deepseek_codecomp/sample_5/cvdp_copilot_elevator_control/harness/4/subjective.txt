The system uses call_requests_internal to maintain an accurate snapshot of active floor requests by creating an internal copy of the input call_requests. When a new request arrives, it is added to call_requests_internal while clearing any previously serviced requests. During movement, the system checks call_requests_internal to process new requests without duplication, updating the current_floor_next accordingly. This mechanism ensures that each request is handled exactly once per cycle, preventing duplicates. During both movement and idle states, the system efficiently processes incoming requests and cleans up handled ones, maintaining optimal performance.
