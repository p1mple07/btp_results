The state machine ensures that changes in the select signals (select_a_i, select_b_i, select_c_i) are ignored during a pending transaction by transitioning to the next state immediately when a new select is detected. Once in the ACCESS state, the machine waits for the current transaction to complete (e.g., when apb_pready_i becomes high or time_out resets). Only after the transaction is complete does the machine accept new selections, thus preventing concurrent events during a pending transaction.
