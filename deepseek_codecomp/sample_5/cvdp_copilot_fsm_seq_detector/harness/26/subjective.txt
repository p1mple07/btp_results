The testbench specification document explains how the testbench maintains a sliding window of the input stream and validates the detection of the sequence `01001110` by DUT in the section titled **4.3 Reference Sequence Detector Logic Using Shift Register**. In this section, it describes that the reference logic uses a shift register to store the last `Length` bits of the input sequence. The shift register is updated on every clock cycle by shifting in the current input bit and discarding the oldest bit. This setup allows the reference logic to continuously check for the target sequence within the sliding window of the input stream.
