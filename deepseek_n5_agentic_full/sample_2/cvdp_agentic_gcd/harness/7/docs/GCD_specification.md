# RTL Specification for GCD Calculation using Stein's Algorithm

This document describes the high-level RTL architecture for a GCD calculator based on Stein's algorithm. The design is separated into three main modules: a top-level module that interconnects the datapath and control path, a control path module for sequencing, and a datapath module for arithmetic and data manipulation.

- **Stein's Algorithm (Binary GCD):**
    Stein’s algorithm or binary GCD algorithm is an algorithm that computes the greatest common divisor of two non-negative integers. Stein’s algorithm replaces division with arithmetic shifts, comparisons, and subtraction. Following is the algorithm to find GCD using Stein’s algorithm gcd(a, b):
    - Base case:
        - If both a and b are 0, gcd is zero gcd(0, 0) = 0.
        - gcd(a, 0) = a and gcd(0, b) = b because everything divides 0.

    - Step 1: Remove Common Factors of 2:
        - If a and b are both even, gcd(a, b) = 2*gcd(a/2, b/2) because 2 is a common divisor. Multiplication with 2 can be done with a bitwise shift operator.

    - Step 2: Handling Even and Odd Cases:
        - If a is even and b is odd, gcd(a, b) = gcd(a/2, b). Similarly, if a is odd and b is even, then gcd(a, b) = gcd(a, b/2). It is because 2 is not a common divisor.

    - Step 3: Handling Odd Numbers:
        - If both a and b are odd, then gcd(a, b) = gcd(|a-b|/2, b). Note that the difference between two odd numbers is even

    - Step 4:Final GCD:
        - Repeat steps 1–3 until a = b, or until a = 0.
        - In either case, the GCD is  2<sup>k</sup> *b, where k is the number of factors of 2 removed in Step 1.

## 1. Top-Level Module: `gcd_top_1`

### Overview
The top-level module orchestrates the GCD calculation by instantiating both the control path and datapath modules. The parameterizable `WIDTH` defines the bit width of the input operands and output.

### Ports
- **Inputs:**
  - `clk`: Clock signal. Design is synced to the posedge of clk.
  - `rst`: Reset signal. Active high.
  - `A [WIDTH-1:0]`: First operand.
  - `B [WIDTH-1:0]`: Second operand.
  - `go`: Start signal for beginning the computation. Active high.
- **Outputs:**
  - `OUT [WIDTH-1:0]`: Output containing the computed GCD.
  - `done`: Indicates the completion of the GCD calculation. Active high.

### Internal Signals
- **`equal`**: A signal computed in the datapath; indicates whether the current values of the operands are equal.
- **`controlpath_state [1:0]`**: Encodes the state of the control path.  
- **`done`**: Generated by the control path when the calculation is finished.

### Instance Connections
- **Control Path Instance (`gcd_controlpath_3`):**  
  - Receives `clk`, `rst`, `go`, and the `equal` signal.
  - Outputs the current state (`controlpath_state`) and the `done` flag.
  
- **Datapath Instance (`gcd_datapath_6`):**  
  - Receives `clk`, `rst`, inputs `A` and `B`, and the current control state (`controlpath_state`).
  - Computes and outputs the `equal` flag and the GCD result (`OUT`).

## 2. Control Path Module: `gcd_controlpath_3`

### Purpose
This module generates control signals based on an internal state machine, dictating when to load new values, continue processing, or finish the computation.

### Ports
- **Inputs:**
  - `clk`: Clock signal. Design is synced to the posedge of clk.
  - `rst`: Reset signal. Active high.
  - `go`: Start signal. Active high.
  - `equal`: Signal from the datapath indicating if both operands are equal. Active high.
- **Outputs:**
  - `controlpath_state [1:0]`: Current state of the control logic.
  - `done`: Computation completion flag. Active high.

### State Machine Description
- **States:**
  - **S0 = 2'd0 (Idle/Load):**  
    When `go` is inactive, the state remains in S0. When active, the state transitions to S2.
  - **S2 = 2'd2 (Processing):**  
    The state loops in S2 while waiting for the operands to become equal. Once `equal` is asserted, the state moves to S1.
  - **S1 = 2'd1 (Completion):**  
    A one-cycle state to signal the end of computation before returning to S0.
- Note : The control path RTL stores the FSM state in `curr_state`  
- **State Transitions:**
  - From **S0**: Remains in S0 if `go` is low; otherwise transitions to S2.
  - From **S2**: If `equal` is true, transitions to S1; otherwise remains in S2.
  - From **S1**: Transitions back to S0.

- **Signal Generation:**
  - The output `done` is asserted 1 clock cycle after entering the state S1.
  - The current state is continuously assigned to `controlpath_state`.

- On reset assertion transition so S0 state and deassert `done`.

## 3. Datapath Module: `gcd_datapath_6`

### Purpose
This module implements the arithmetic operations of Stein's GCD algorithm. It conditionally shifts and subtracts the input operands based on the algorithm until the final GCD is computed.

### Parameter
- **`WIDTH`**: Bit width of the input operands (and the output).

### Ports
- **Inputs:**
  - `clk`: Clock signal. Design is synced to the posedge of clk.
  - `rst`: Reset signal. Active high.
  - `A [WIDTH-1:0]`: First operand.
  - `B [WIDTH-1:0]`: Second operand.
  - `controlpath_state [1:0]`: State signal from the control module.
- **Outputs:**
  - `equal`: A flag indicating if the two operands (after internal updates) are equal. Active high.
  - `OUT [WIDTH-1:0]`: The computed greatest common divisor.

### Internal Signals and Registers
- **Registered Data:**
  - `A_ff [WIDTH-1:0]`, `B_ff [WIDTH-1:0]`: Register copies of the operands.
  - `OUT_ff [WIDTH-1:0]`: Output register for the GCD result.
  - `k_ff`: Register used to count the number of common factors of 2 (shifts).
  
- **Next-State Signals:**
  - `next_A_ff, next_B_ff, next_OUT`: Next values computed for `A_ff`, `B_ff`, and `OUT_ff`.
  - `next_k_ff`: Next value for `k_ff`.

- **Computational Signals:**
  - `diff`: Holds the difference result used when subtracting the operands.
  - `gcd_val`: Intermediate computed GCD value.
  - `a_even`: Indicates if `A_ff` is even.
  - `b_even`: Indicates if `B_ff` is even.
  - `both_even`: Indicates if both `A_ff` and `B_ff` are even.

### Datapath Control Based on `controlpath_state`
- **State S0 (Load / Initialization):**
  - Loads input values (`A` and `B`) into the registers.
  - Resets the factor counter `k_ff` and the output register.
  
- **State S2 (Processing / Iterative Reduction):**
  - **Both Operands Even:**  
    If both `A_ff` and `B_ff` are even, both values are shifted right (divided by 2), and `k_ff` is incremented.
  - **One Operand Even:**  
    If one operand is even, only the even operand is shifted right.
  - **Both Operands Odd:**  
    When both operands are odd, subtract the smaller from the larger, shift the difference right, and reassign the registers.
  - **Edge Cases:**  
    If one of the registers becomes 0, the module redirects values to converge on the non-zero operand.
  
- **State S1 (Completion / Final Computation):**
  - When the operands become equal, perform final computation:
    - If both registers are zero, `gcd_val` is set to 0.
    - Otherwise, the non-zero operand is combined with the shift count `k_ff` (the common factors of 2) to produce the final result.
  - The computed GCD (`gcd_val`) is then saved to `OUT_ff`.

### Data Path Signal Propagation
- The updated values (`A_ff`, `B_ff`, `k_ff`, and `OUT_ff`) are registered at every positive edge of the clock. On reset, all these registers are cleared.
