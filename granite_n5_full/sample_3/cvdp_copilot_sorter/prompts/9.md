Sorting should arrange the elements of the array in ascending order, such that the smallest element is at index 0 and the largest element is at index N−1.

**Brick Sort** (also known as **Odd-Even Sort**) is a simple comparison-based sorting algorithm that repeatedly alternates between comparing/swapping all “odd-even” index pairs and all “even-odd” index pairs. After each pass of pairwise comparisons, larger elements gradually “bubble” to the right and smaller elements “bubble” to the left, ensuring that enough iterations will fully sort the array. On **even-numbered passes**, the module compares and, if needed, swaps **even-odd** index pairs ((0,1), (2,3)) (compares elements at even-indexed positions with their immediate next neighbors). On **odd-numbered passes**, it compares and, if needed, swaps **odd-even** index pairs ((1,2), (3,4)) (compares elements at odd-indexed positions with their immediate next neighbors). Each pass is processed in multiple clock cycles: each clock cycle steps through the relevant pairs, performing one compare-and-swap per cycle. After finishing \(N\) passes, the array is guaranteed to be sorted.

**Algorithm Example:**  
Suppose we have the array \([14,12,13,15]\):

1. **Iteration 0, Even-Odd Pass**  
   - Compare indexes 0,1: \((15,13)\) → swap → \([14, 12, 15, 13]\)  
   - Compare indexes 2,3: \((12,14)\) → no swap → \([14, 12, 15, 13]\)

2. **Iteration 1, Odd-Even Pass**  
   - Compare indexes 1,2: \((15,12)\) → swap → \([14, 15, 12, 13]\)  

3. **Iteration 2, Even-Odd Pass**  
   - Compare indexes 0,1: \((13,12)\) → swap → \([14, 15, 13, 12]\)  
   - Compare indexes 2,3: \((15,14)\) → swap → \([15, 14, 13, 12]\)

4. **Iteration 3, Odd-Even Pass**  
   - Compare indexes 1,2: \((13,14)\) → no swap → \([15, 14, 13, 12]\)  

Now \([15,14,13,12]\) is sorted. It runs \(N\) passes to guarantee sorting.
