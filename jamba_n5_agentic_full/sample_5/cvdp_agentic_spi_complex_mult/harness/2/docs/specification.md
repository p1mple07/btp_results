# SPI Slave Complex Multiplication Specification

## Overview
The `spi_complex_mult` module implements a SPI Slave module that receives the complex number components Ar, Ai, Br, and Bi (real and imaginary parts) via SPI and performs complex multiplication using DSP operations. The results are stored in internal registers and are transmitted back through SPI. Additionally, the module supports SPI Mode 0 (CPOL=0, CPHA=0).

## Features
- Receives the complex number components Ar, Ai, Br, and Bi by SPI.
- Implements a complex multiplication using DSP operations.
- The results are stored in internal registers and transmitted back through SPI.
- While transmitting the result of the complex multiplication, the system can simultaneously receive data for the next multiplication.
- Operates in SPI Mode 0 (CPOL=0, CPHA=0), where data is sampled on the rising edge and transmitted on the falling edge.

## Description of the SPI Protocol (Serial Peripheral Interface)

The **Serial Peripheral Interface (SPI)** is a **high-speed, full-duplex, synchronous** serial communication protocol used to exchange data between a **master device** and one or more **slave devices**. SPI follows a **master-slave architecture**, where:
- **The master** controls the communication, generates the clock signal (`spi_sck`), and selects which slave to communicate with.
- **The slave(s)** respond to the master's requests but do not initiate communication.

SPI transfers data using a **synchronous serial clock (spi_sck)**, allowing data to be **sent and received simultaneously (full-duplex)**. 

### SPI Data Transfer Process
1. **Master selects the slave** by pulling `spi_cs_n` low (`0`).
2. **Master generates clock pulses (`spi_sck`)** to synchronize the transfer.
3. **Master sends data via `spi_mosi`**, and the slave **sends data back via `spi_miso`** (if needed).
4. **Master reads data on `spi_miso`** while transmitting on `spi_mosi`.
5. **After the transaction**, the master **deactivates the slave (`spi_cs_n` = `1`)**.

### SPI Modes (Clock Configuration)
The SPI protocol has **four modes**, controlled by **two bits:**
- **CPOL (Clock Polarity)**: Determines the idle state of the clock.
- **CPHA (Clock Phase)**: Determines when data is sampled.

| **Mode** | **CPOL** | **CPHA** | **Clock Idle State** | **Data Captured On** | **Data Changed On** |
|----------|----------|----------|----------------------|----------------------|---------------------|
| **0**    | 0        | 0        | Low (`0`)            | Rising edge          | Falling edge        |
| **1**    | 0        | 1        | Low (`0`)            | Falling edge         | Rising edge         |
| **2**    | 1        | 0        | High (`1`)           | Falling edge         | Rising edge         |
| **3**    | 1        | 1        | High (`1`)           | Rising edge          | Falling edge        |

Master and slave must operate in the same mode to ensure proper data synchronization.
