The testbench verifies the correct behavior of the compression_engine module by comparing its output signals (mantissa_o and exponent_o) against expected values computed using the compute_expected task. The compute_expected task scans the input vector num_i from the highest bit (bit 23) downward to identify the first set bit among the high-order bits (bits 23 to 12), thereby forming a one-hot encoding for the exponent. It then converts this one-hot encoding into a binary value by iterating through the bits and assigning the corresponding binary number, adding 1 if any high-order bit is set to compute the expected exponent. Based on whether a high-order bit is set, the task extracts the mantissa either from the lower 12 bits of num_i or from a 12-bit segment starting at an index determined by the computed exponent, ensuring that the expected outputs precisely match the module's behavior for accurate verification.
