The testbench instantiates two versions of the DUT, one configured for binary encoding and the other for one-hot encoding, ensuring that both output formats are verified under identical input conditions. It applies a series of test vectors that cover a range of scenarios including no set bits, a single bit set, multiple bits set, and all bits set, which thoroughly exercises the module's logic. The testbench waits for the output signals (Out_Valid) to become active, thereby accounting for any pipeline delays before comparing the results. It calculates the expected first set bit index and the corresponding one-hot encoding using dedicated functions and then compares these expected values with the DUT outputs (Out_FirstBit, Out_Found, and Out_Valid) for both formats. If there is a discrepancy between the expected and actual outputs, the testbench displays error messages, ensuring that any malfunction in detecting the correct bit position is immediately flagged. Finally, the testbench handles edge cases such as inputs with no set bits and inputs with all bits set, confirming that the module correctly identifies the first set bit or appropriately indicates when no set bits are found.
