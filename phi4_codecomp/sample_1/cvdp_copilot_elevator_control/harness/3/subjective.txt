The FSM in the IDLE state determines the movement direction by comparing the current floor (current_floor_reg) with the highest active request (max_request) and the lowest active request (min_request). If max_request is greater than the current floor, the FSM transitions to MOVING_UP, ensuring that the elevator moves toward the furthest upward request. Otherwise, if min_request is less than the current floor, it transitions to MOVING_DOWN, thereby prioritizing the downward direction. This logic efficiently responds to active requests by committing to a single direction per cycle and avoiding unnecessary oscillations, as the elevator only re-evaluates its state in the IDLE state after serving a request or upon reset.
