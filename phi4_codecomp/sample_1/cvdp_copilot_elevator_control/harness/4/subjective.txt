The system uses call_requests_internal to capture and store all active external floor requests by merging the input call_requests with its current state, ensuring that any new request is immediately registered. It integrates new requests by performing an OR operation with the existing internal requests, thereby adding any new call to the active set. The system clears serviced requests by checking if a request at the current floor is active and then resetting that bit in the internal register, which prevents duplicate stops at the same floor. This continuous update mechanism ensures that the internal register always reflects the most current request status, avoiding redundant stops. During movement, the updated internal register provides an accurate snapshot of active requests, allowing the elevator to determine the next floor without unnecessary delays. In idle states, the mechanism swiftly detects new requests, enabling the elevator to transition promptly from idle to moving, thus ensuring efficient and timely request handling.
