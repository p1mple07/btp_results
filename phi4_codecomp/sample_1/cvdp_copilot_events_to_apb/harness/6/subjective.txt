The state machine only initiates a transaction when in the IDLE state by latching the address and data from the select signals and transitioning to the SETUP state. Once in the SETUP or ACCESS states, the next state logic ignores any changes in the select signals, ensuring that the ongoing transaction is not interrupted. Only after the transaction is completed and the state returns to IDLE does the design resume monitoring the select signals, thereby preventing events triggered during a previous transaction from being captured.
