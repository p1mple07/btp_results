The testbench specification document explains the sliding window mechanism and sequence validation in Section 4.3, titled 'Reference Sequence Detector Logic Using Shift Register'. In this section, it is described that the testbench maintains a sliding window of the input stream using a shift register (seq_shift_reg) that stores the last 'Length' bits. On every clock cycle, the shift register is updated by shifting in the current input bit (seq_in) and discarding the oldest bit, ensuring that the register always contains the most recent 'Length' bits. This updated window is then compared with the informed sequence (01001110) to validate whether the sequence has been detected by the DUT.
