The RTL extracts each mapping index from the concatenated mapping_in signal and, in the always_ff block, checks if the index is within the valid range of 0 to N-1; if it is, the data is routed accordingly from data_in to data_out based on the config_in setting. When a mapping index exceeds N, the corresponding output lane is assigned a '0', effectively ignoring that out-of-range index. Although the mapping_in bus is wide enough to allow out-of-range values due to its bit-width determined by $clog2(N), such indices are feasible to input but simply result in a no-operation for the affected data lane.
