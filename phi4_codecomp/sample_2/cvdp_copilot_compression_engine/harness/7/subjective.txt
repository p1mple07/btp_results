The compute_expected task begins by scanning the high-order bits of the input vector to locate the first set bit, which it then uses to create a one‐hot encoded representation of the exponent. It converts this one‐hot encoding to a binary value by iterating over the bits and assigning the corresponding index, adding one to the result if any bit is set to match the RTL logic. For the mantissa, the task computes the starting bit position based on the computed exponent and extracts the subsequent 12 bits from the input vector when the one‐hot encoding is active, or simply uses the lower 12 bits if no set bit is found. This systematic computation of expected exponent and mantissa values enables the testbench to accurately compare the UUT outputs with the expected results, thereby verifying the correct behavior of the compression_engine module when the input vector contains multiple set bits.
