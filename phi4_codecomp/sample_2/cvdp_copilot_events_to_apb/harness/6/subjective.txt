The state machine starts in the IDLE state where it checks the select signals to initiate a transaction, ensuring that only one transaction is started at a time. Once a transaction begins, the state machine immediately transitions to the SETUP and then to the ACCESS state, during which the select signals are not evaluated, meaning any changes to them are ignored. This design prevents overlapping transactions by ensuring that any new select events triggered during an ongoing transaction do not interfere until the current transaction completes and the state machine returns to IDLE.
