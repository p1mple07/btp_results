The sliding window mechanism and sequence validation are explained in Section 4.3, 'Reference Sequence Detector Logic Using Shift Register.' In this section, the document describes how the testbench maintains a sliding window of the input stream using a shift register (seq_shift_reg) that is updated on every clock cycle by shifting in the current input bit and discarding the oldest bit. This mechanism ensures that the shift register always contains the most recent 'Length' bits of the input sequence. The reference logic then compares this sliding window to the informed sequence (01001110) in a combinational block, thereby validating whether the DUT has correctly detected the sequence.
