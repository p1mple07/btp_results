The testbench uses the apb_write task to send the value 0xF0 to the Direction Control Register (address 0x1C), thereby configuring GPIO[7:4] as outputs and GPIO[3:0] as inputs. It then employs the apb_read task to retrieve the register value from the DUT and compares it to the expected 0xF0 value to verify that the configuration was correctly applied. This read-back mechanism confirms that the DUT correctly processes the APB write transaction and updates the internal state of the GPIO direction register. By ensuring that both input and output configurations are correctly set, the testbench validates the proper interaction between the APB interface tasks and the DUT's GPIO control logic.
