The testbench instantiates two versions of the DUT, one configured for binary encoding and the other for one-hot encoding, allowing simultaneous verification of both output formats. It applies a comprehensive set of test vectors using the apply_test_vector task, covering conditions such as no set bits, a single set bit, multiple set bits, and various specific bit positions. The testbench waits for the output valid signals (Out_Valid_Binary and Out_Valid_OneHot) before comparing the DUT outputs to the expected results, thereby accounting for the inherent pipeline delays. For each test vector, the expected first set bit index is computed using a dedicated function and then converted into one-hot encoding, which is compared against the DUT’s outputs. The testbench displays the results and flags errors if the DUT’s outputs do not match the expected binary index or one-hot encoding, ensuring that both encoding formats are correctly implemented. By handling edge cases like no set bits, all bits set, and specific bit positions, the testbench thoroughly validates the functionality of the cvdp_copilot_decode_firstbit module under a variety of input conditions.
