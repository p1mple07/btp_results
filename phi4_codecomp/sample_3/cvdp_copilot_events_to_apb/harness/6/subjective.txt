The state machine ensures that any events triggered with changes in the select signals are ignored while a previous transaction is in progress by only allowing a state transition from IDLE to SETUP when no transaction is active. Once the state machine enters the SETUP state, it immediately moves to the ACCESS state and remains there until the peripheral signals readiness or a timeout occurs, thereby disregarding any new select signal events. This design prevents overlapping transactions by using a synchronous state transition mechanism along with a timeout counter to ensure that only one transaction is processed at a time.
