Section 4.3, 'Reference Sequence Detector Logic Using Shift Register', explains that the testbench maintains a sliding window of the input stream by using a shift register (seq_shift_reg) that is updated on every clock cycle. This register holds the most recent 'Length' bits of the input, and the always_comb block then compares this sliding window with the informed sequence (01001110) to validate the detection of the sequence by the DUT.
