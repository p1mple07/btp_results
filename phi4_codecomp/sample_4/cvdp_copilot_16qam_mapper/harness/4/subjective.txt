The Golden Model is implemented in the 'model_system' task within the testbench (verif/tb_qam16_mapper_interpolated.sv). In this task, the testbench maps each binary input symbol to its corresponding in-phase (I) and quadrature (Q) values using case statements, and then it computes the interpolated values between successive mapped symbols. Finally, it builds the expected output arrays (model_I and model_Q) that serve as the unbiased reference model for verifying the DUT's behavior.
