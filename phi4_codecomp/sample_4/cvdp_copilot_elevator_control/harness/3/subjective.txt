In the IDLE state, the FSM first checks if there are any call requests and then compares the current floor (current_floor_reg) with the highest (max_request) and lowest (min_request) requested floors. If the highest requested floor is above the current floor, the FSM transitions to MOVING_UP, thereby prioritizing upward movement when both upward and downward requests are present. Otherwise, if the highest request is not above the current floor but the lowest request is below it, the FSM transitions to MOVING_DOWN. This prioritization mechanism, combined with state transitions that maintain the chosen direction until the request is fulfilled, ensures efficient handling of requests and prevents unnecessary direction changes.
