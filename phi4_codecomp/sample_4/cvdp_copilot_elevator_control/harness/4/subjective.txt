The system integrates new floor requests by OR-ing the external call_requests input with the existing call_requests_internal register, ensuring that any new request is added to the active list. This integration preserves previously active requests while allowing the system to accumulate multiple floor requests simultaneously. When the elevator reaches a floor, the corresponding bit in call_requests_internal is cleared if it matches the current floor, thereby preventing duplicate stops. This clearing mechanism ensures that once a request has been serviced, it does not trigger another stop at the same floor. By continuously updating call_requests_internal during both movement and idle states, the system accurately tracks active requests and computes the highest and lowest floor requests efficiently. This mechanism optimizes route planning and reduces unnecessary stops, ensuring smooth and efficient elevator operation.
