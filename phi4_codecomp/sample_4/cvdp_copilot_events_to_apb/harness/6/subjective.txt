The state machine uses a current_state signal to track the progress of a transaction. When the design is in the SETUP or ACCESS states, the combinational logic does not update the transaction based on changes to the select signals, effectively ignoring any new events. This ensures that once a transaction is in progress, any subsequent events triggered by changes in select_a_i, select_b_i, or select_c_i are disregarded until the state returns to IDLE.
