The first block is the always block that calculates the expected output value. In this block, based on the current mode (using a case statement), the testbench computes the expected value for serial_out from serial_in and internal signals (such as tb_clk_pulse, tb_prev_serial_in, tb_alt_invert_state, tb_parity_out, and tb_counter). The second block is the function check_output, which takes the computed expected value, the corresponding feature name, and the iteration number, and then compares it with the actual serial_out from the DUT. It logs a PASS message if they match or an ERROR message with details (expected value, actual value, mode, iteration, and simulation time) if they do not.
