The elevator control system uses call_requests_internal to maintain an up-to-date record of active floor requests by integrating new inputs from the external call_requests signal. It updates call_requests_internal on every clock cycle by ORing it with the external call_requests, ensuring that any new request is promptly added to the active list. When the elevator reaches a floor, the corresponding bit in call_requests_internal is cleared, which prevents the system from stopping at that floor again. This clearing mechanism avoids duplicate stops by ensuring that each request is served only once. During movement, the system continuously evaluates call_requests_internal to determine the highest or lowest pending request, enabling efficient directional navigation. In idle state, the accurate reflection of active requests in call_requests_internal allows the system to quickly transition to movement, thereby optimizing request handling and minimizing unnecessary stops.
