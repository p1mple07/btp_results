The state machine only processes the select signals when it is in the IDLE state, so any changes in select_a_i, select_b_i, or select_c_i during SETUP or ACCESS are ignored. While in the SETUP or ACCESS states, the controller continues its current transaction without re-evaluating the select signals, ensuring that a new event does not override the ongoing operation. This design guarantees that only transactions initiated when the controller is idle are captured, preventing any interference from events triggered during an active transaction.
