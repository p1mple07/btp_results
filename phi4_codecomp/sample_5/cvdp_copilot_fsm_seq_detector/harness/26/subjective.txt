The testbench specification document explains this in Section 4.3, 'Reference Sequence Detector Logic Using Shift Register'. In that section, it describes how the testbench maintains a sliding window of the input stream by using a shift register that holds the most recent 'Length' bits of the input. Each clock cycle, the shift register is updated by shifting in the current bit and discarding the oldest one. The reference logic then compares the contents of this sliding window (the shift register) with the informed sequence (01001110) to validate whether the sequence has been detected. This mechanism is directly implemented in the provided code snippet, where the first always block updates the shift register and the second always_comb block performs the comparison to set the reference detection flag.
