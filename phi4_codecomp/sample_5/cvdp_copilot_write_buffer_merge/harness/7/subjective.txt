The testbench uses two counters: one (num_inputs) tracks the total number of input write events, which is compared against num_inputs_expected (calculated as num_outputs_expected multiplied by BUFFER_DEPTH), and the other (num_wr_en_out) counts the output write events. The simulation loop continuously checks these counters and ends the simulation once the number of output writes (num_wr_en_out) reaches the expected count (num_outputs_expected), ensuring that both input and output operations have been completed.
