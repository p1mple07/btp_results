## Functional Description

### Finite State Machine (FSM)

The FSM manages transitions between states based on the input signals and the current state. The logic for determining the next state is implemented combinationally, while the actual state transition occurs sequentially on the rising clock edge.

### States and Transitions

1. **`IDLE`**:
   - Waits for the `write_en` signal.
   - Transitions to `ANALYZE` if `write_en` is asserted, otherwise remains in `IDLE`.

2. **`ANALYZE`**:
   - Checks the Most Significant Bit (MSB) of `wdata`.
   - Transitions to `XOR_DATA` if the MSB of `wdata` is 1, otherwise transitions to `WRITE`.

3. **`XOR_DATA`**:
   - Performs the XOR operation on `wdata`.
   - Always transitions to `WRITE`.

4. **`WRITE`**:
   - Writes the processed data to the appropriate register.
   - Transitions to `IDLE` if `write_en` is asserted again, otherwise transitions to `LOST`.

5. **`LOST`**:
   - Represents a state where no consecutive write operation occurs.
   - Transitions to `CHECK_KEY` if `read_en` is asserted; otherwise remains in `LOST`.

6. **`CHECK_KEY`**:
   - Checks the validity of the HMAC key.
   - Transitions to `TRIG_WAIT` if the key is valid; otherwise transitions back to `WRITE`.

7. **`TRIG_WAIT`**:
   - Waits for the `i_wait_en` signal to be de-asserted, otherwise, remains on the same state.
   - If both `hmac_data` and `hmac_key` are non-zero, transitions to `IDLE`.
   - Otherwise, transitions back to `WRITE`.

### HMAC Key Validation (Combinational Logic)
1. **Pattern Check**:  
   - 2 MSB and 2 LSB of the key are verified against expected values (zeros).
   - If the pattern matches, the key is valid.
2. **Error Flag**:  
   - If the key is valid, no error is flagged.  
   - If the key format is incorrect, an error flag is raised.

### Data Processing
- Data processing applies a transformation to `wdata` in the `XOR_DATA` state. It performs an XOR operation with a pair of `2'b01` for every 2 bits of `wdata`.

### Sequential Write Logic
- Data is written to registers during the `WRITE` state.
- Special handling for HMAC key and data registers at addresses `0` and `1`.
- `hmac_valid` should be asserted only during `WRITE` state.

### Sequential Read Logic
- Data is read from registers if `read_en` is active and FSM is not in the `WRITE` state.
- Special handling for HMAC key and data registers at addresses `0` and `1`.

## Reset Behavior
- On reset (`rst_n` low), `hmac_key` and `hmac_data` are reset to 0, the outputs `r_data` and `hmac_valid` are cleared to zero and the State of FSM returns to IDLE.

## Edge Cases:
- `write_en` and `read_en` should not be asserted at the same cycle, therefore if at cycle 1 write_en = 1, then read_en = 0, and vice-versa.

## Latency

- **Write Operation:**
  - The latency is four clock cycles from the first rising edge of `write_en`.

- **Read Operation:**
  - The latency is one clock cycle as the read operation is not directly tied to the FSM.
  - However, the read operation is only performed when `write_en` is deasserted.
